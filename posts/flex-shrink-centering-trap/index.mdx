### 事情的起因

我在给博客编辑器做一个图片管理轨道 —— 就是一个横向滚动的图片缩略图条，左边有个上传按钮，图片可以左右滑动浏览。

结构大概是这样的：三层嵌套。下面左边是 bug 状态（外壳被 flex-shrink 压扁），右边是修复后的效果：

### 上手段：getBoundingClientRect

既然肉眼看不出来，那就让数字说话。

我在组件里加了一段临时调试代码，用 `getBoundingClientRect()` 把每一层的实际尺寸都打出来：

```typescript
useEffect(() => {
  if (!railRef.current) return;
  const outer = railRef.current.closest('[style*="overflow: hidden"]');
  const rail = railRef.current;
  const innerFlex = rail.firstElementChild;
  const firstItem = innerFlex?.firstElementChild;

  console.table({
    outer: {
      top: outer?.getBoundingClientRect().top,
      height: outer?.getBoundingClientRect().height,

    rail: {
      top: rail.getBoundingClientRect().top,

    innerFlex: {
      top: innerFlex?.getBoundingClientRect().top,

      alignItems: getComputedStyle(innerFlex!).alignItems,
    },
    firstItem: {
      top: firstItem?.getBoundingClientRect().top,
      height: firstItem?.getBoundingClientRect().height,
```

外壳高度 **93.125px**？？？:sticker[v2_cc91a343-c1d3-489f-b151-3d63f947e70l.gif]:

轨道内部 inline-flex 容器高度是 104px（`6.5rem`，正确），但外壳高度只有 93.125px。

外壳比里面矮了将近 11px。`overflow: hidden` 把底部裁掉了，所以看起来图片「贴着底部」—— 其实图片确实居中了，只不过外壳太矮，把上面的空间吃掉了。

**问题根本不在 `align-items` 上。** :sticker[v2_9e96d7cd-c2a9-4b9f-b51b-1dafc8ea940l.gif]:

### 真凶：flex-shrink

外壳为什么会从 104px 被压到 93.125px？

往上看一层。这个图片管理轨道被放在编辑器的主区域里，结构大概是：

```html
<div style="flex: 1; display: flex; flex-direction: column;">
  <PostImageManager />   <!-- 图片轨道 -->
  <MdxEditor />          <!-- 编辑器，flex: 1 -->
</div>
```

父容器是 `flex-direction: column`。编辑器用了 `flex: 1` 吃掉所有剩余空间。

关键来了：**flex 子元素的默认 `flex-shrink` 是 `1`。**

当父容器空间不够时（编辑器想要尽可能多的空间），flex 布局会按 `flex-shrink` 比例压缩子元素。图片轨道没有设 `flex-shrink: 0`，所以它被压缩了。

从 104px 压到 93.125px，刚好少了大约 10%。

而 `overflow: hidden` 忠实地把溢出部分裁掉了。内部的 inline-flex 容器不知道外壳被压缩了，它还是按 104px 的高度做居中计算。结果就是：居中是居中了，但外壳装不下，视觉上看起来就是贴着底部。

### 修复

一行 CSS：

```css
flex-shrink: 0
```

加在外壳上。:sticker[getimgdata-5.jpg]:

```tsx
<div style={{
  position: 'relative',
  overflow: 'hidden',
  flexShrink: 0,        // ← 就这一行
  borderRadius: '2rem',
  // ...
}}>
```

加完之后，外壳高度回到了正确的值，图片完美居中。

### 为什么这个 bug 这么难找

1. **症状和原因完全不在一个层级。** 你看到的是「`align-items: center` 不生效」，但问题出在祖父级的 flex 布局上。光盯着 `align-items` 这一层看，怎么都看不出来。

2. **93.125px 这个数字太诡异了。** 它不是任何你写过的值，是 flex 布局算出来的。不拿 `getBoundingClientRect()` 量一下，很难意识到容器被压缩了。

3. **`overflow: hidden` 是帮凶。** 它把溢出裁掉了，让你看不到内部其实是正常的。没有它的话，至少能看到内容溢出，从而意识到容器尺寸不对。

4. **`flex-shrink: 1` 是默认值。** 你没写它，它就在那里。大多数时候它挺好的（让布局自适应），但碰上固定高度的组件，就容易出这种幺蛾子。

### 经验

固定尺寸的 flex 子元素（图片轨道、工具栏、状态栏这类东西），顺手加个 `flex-shrink: 0` 大概率能省不少事。

这不是什么高深的知识，但当你在一个复杂的嵌套布局里 debug 的时候，你的注意力全在出问题的那一层，根本不会想到去检查三层之外的 flex 容器。

所以下次 CSS 表现离谱的时候，`getBoundingClientRect()` 量一下实际尺寸吧，说不定真相就藏在某个你压根没碰过的祖先节点里。:sticker[getimgdata-7.gif]:
