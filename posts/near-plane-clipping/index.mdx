### 透视投影的公式

把一个 3D 点 $(x, y, z)$ 投影到 2D 屏幕上，透视投影的核心公式是：

$
x' = x \cdot \frac{d}{d + z}, \quad y' = y \cdot \frac{d}{d + z}
$

$d$ 是相机到投影平面的距离。这个公式做的事情很直觉：离相机越远（$z$ 越大），缩放因子 $\frac{d}{d+z}$ 越小，物体看起来越小。这就是近大远小。

### 当 z &lt; -d 的时候

问题来了。在这个坐标系里，投影面在 $z = 0$，相机在投影面后方 $d$ 的位置，也就是 $z = -d$。如果一个点的 $z$ 坐标小于 $-d$，它就穿过了相机，跑到相机后面去了：

$
d + z < 0 \implies \frac{d}{d + z} < 0
$

缩放因子变成负数了。$x'$ 和 $y'$ 的符号翻转，这个点会被投影到屏幕的**对称位置**。本来在右上角的点，突然跳到左下角。

如果一个矩形的四个角里有一部分在相机前面、一部分在相机后面，投影出来的四边形会变成一个扭曲的蝴蝶结形状，或者直接翻转成一个巨大的覆盖整个屏幕的多边形。

拖动滑块，让矩形从相机前方穿到后方。左边是俯视图（3D 空间中的位置），右边是屏幕上的投影结果。注意矩形部分穿过 $z = -d$ 时的形状变化：

这就是为什么需要**近平面裁剪**（near-plane clipping）。

### 正交投影 vs 透视投影

在讨论裁剪方案之前，先看看为什么只有透视投影需要裁剪。

$
\text{正交：} \quad x' = x \cdot \text{zoom}, \quad y' = y \cdot \text{zoom}
$

$
\text{透视：} \quad x' = x \cdot \frac{d}{d + z}, \quad y' = y \cdot \frac{d}{d + z}
$

正交投影的缩放因子是常数，不依赖 $z$ 坐标。不管点在相机前面还是后面，投影坐标都是有限的，不会爆炸。所以正交投影不需要裁剪。

拖动滑块对比两种投影方式。注意右侧的屏幕预览——正交投影的投影大小始终不变，而透视投影会随距离变化，超出屏幕范围时只能显示一部分：

### GPU 的标准做法

在标准的 3D 渲染管线里（OpenGL / Vulkan / DirectX），近平面裁剪是硬件自动完成的。渲染管线定义了一个视锥体（frustum），由 6 个平面围成：近平面、远平面、上下左右。任何超出视锥体的几何体都会被裁剪。

具体来说，GPU 在顶点着色器之后、光栅化之前做裁剪。如果一个三角形的某些顶点在近平面外面，GPU 会沿着近平面把三角形切开，只保留在视锥体内的部分。切出来的可能是一个三角形，也可能是一个四边形（再拆成两个三角形）。

这个过程用的是 **Sutherland-Hodgman 裁剪算法**，图形学的经典算法之一。

拖动滑块，看看三角形穿过近平面时 Sutherland-Hodgman 算法怎么裁剪。紫色虚线是原始三角形，绿色实线是裁剪后保留的部分，红色圆点是裁剪产生的新顶点：

### Sutherland-Hodgman 算法实现

核心思路是：对多边形的每条边，检查两个端点是否在裁剪面同侧。如果一个在前一个在后，算出交点；保留前面的端点和交点，丢弃后面的端点。

```typescript
function clipPolygonByNearPlane(
  vertices: [number, number, number][],
  nearZ: number
): [number, number, number][] {
  const output: [number, number, number][] = [];
  const n = vertices.length;

  for (let i = 0; i < n; i++) {
    const curr = vertices[i];
    const next = vertices[(i + 1) % n];
    const currInside = curr[2] >= nearZ;
    const nextInside = next[2] >= nearZ;

    if (currInside) output.push(curr);

    if (currInside !== nextInside) {
      // edge crosses near plane, compute intersection
      const t = (nearZ - curr[2]) / (next[2] - curr[2]);
      output.push([
        curr[0] + t * (next[0] - curr[0]),
        curr[1] + t * (next[1] - curr[1]),
        nearZ,
      ]);
    }
  }
  return output;
}
```

裁剪后的多边形可能有 3~5 个顶点（原始四边形被切掉一个角或两个角），然后对每个顶点做透视投影就行了。

### 实际案例：Canvas 2D 里的近平面裁剪

上面讲的是通用图形学知识。接下来看一个实际项目里的工程取舍。

我在做一个 LVGL 运行时状态可视化工具 [uinspy](https://github.com/W-Mai/uinspy)，其中有一个 Canvas 2D 手写的 3D 渲染器（详见 [一个 3D 渲染器的三次重写](/blog/uinspy-renderer-evolution)）。这个渲染器需要把几百个平行矩形（UI 层）做透视投影画到 Canvas 上。

Canvas 2D 没有 GPU 那样的自动裁剪管线，得自己处理近平面问题。

#### 最简单的方案：整层丢弃

```typescript
// near-plane clip: discard points behind camera
const p = PERSPECTIVE_DISTANCE;
if (persp > 0 && z < -p * 0.95) return null;
```

如果一个点的相机空间 Z 坐标小于 $-p \times 0.95$（接近或超过相机位置），`project` 函数直接返回 `null`。调用方看到 `null` 就跳过这个层不画。

注意 `persp > 0` 这个条件——正交模式下不做裁剪，因为正交投影不存在奇点问题（不需要计算透视变换，不存在爆炸:sticker[zhiyin.gif]:）。

```typescript
// in the render loop
const pts = [
  this.project(l.x, l.y, l.depth),
  this.project(l.x + l.w, l.y, l.depth),
  this.project(l.x + l.w, l.y + l.h, l.depth),
  this.project(l.x, l.y + l.h, l.depth),
];
// if ANY corner is behind near plane, skip entire layer
if (pts.some(p => !p)) return;
```

四个角只要有一个返回 `null`，整个层就不画了。

#### 为什么是 0.95 而不是 1.0

当 $z = -d$ 的时候，$\frac{d}{d+z} \to \infty$，投影坐标会爆到天文数字。留 5% 的余量，在点还没到达奇点之前就丢掉，避免数值爆炸:sticker[zhiyin.gif]:。

太大（比如 $0.99$）的话，接近奇点时投影坐标非常大，Canvas 2D 画一个几万像素的多边形会有性能问题。太小（比如 $0.5$）的话，裁剪太激进，正常视角下也会有层消失。$0.95$ 是一个平衡点。

> 👆 AI 说的。

#### 为什么不做精确裁剪

精确裁剪的意思是：如果一个矩形有两个角在近平面前面、两个角在后面，沿着近平面把矩形切开，只画前面的部分。

在 GPU 管线里这是标准操作（上面的 Sutherland-Hodgman 算法），但在 Canvas 2D 里实现起来比较麻烦：

1. **要算交点：** 矩形的四条边和近平面的交点，需要线性插值
2. **裁剪后的形状不是矩形：** 可能是三角形、五边形，Canvas 2D 的 `fillRect` 用不了，得用 `beginPath` + `lineTo` 画任意多边形
3. **纹理映射更复杂：** 如果层上有 buffer 图像，裁剪后的纹理坐标也要跟着插值

对于 uinspy 的场景，精确裁剪的收益很小——近平面裁剪只在极端旋转角度下才会触发：用户把场景转到几乎平视的时候，最近的几个层可能会穿过相机。（其实就是怕麻烦，丢就丢吧，爱看不看 :sticker[getimgdata-16.jpg]:)

#### Buffer 图像的裁剪

除了层矩形，uinspy 还要渲染 buffer 图像（LVGL display 的屏幕截图）。buffer 图像用 16×16 的小格子做仿射近似（详见 [一个 3D 渲染器的三次重写](/blog/uinspy-renderer-evolution/#canvas-2d-的一个-hackbuffer-图像的透视变形)）。

每个小格子的三个角点都会调用 `project`，如果任何一个角点返回 `null`，这个格子就跳过不画：

```typescript
const p00 = this.project(sw * u0, sh * v0, depth);
const p10 = this.project(sw * u1, sh * v0, depth);
const p01 = this.project(sw * u0, sh * v1, depth);
if (!p00 || !p10 || !p01) continue;  // skip this tile
```

因为格子很小（1/16 的图像大小），跳过一个格子在视觉上几乎看不出来。这比对整张图像做裁剪简单得多，效果也够好。

### 总结

近平面裁剪解决的是透视投影的数学奇点问题：当 3D 点接近或穿过相机位置时，投影坐标会翻转或趋向无穷大。

- **GPU 管线**自动用 Sutherland-Hodgman 算法做精确裁剪
- **Canvas 2D** 没有自动裁剪，需要手动处理。最简单的方案是检测到近平面后面就丢弃，对于大多数场景够用
- **正交投影**不存在这个问题，缩放因子是常数
