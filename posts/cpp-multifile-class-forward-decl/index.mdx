### C++多文件类 未定义类的引用


#### 缘起
有时候因为类之间关系调用比较复杂，会出现很神奇的现象
```cpp
class One{
    int val;
    void fun1(Two two);
};

class Two{
    somecodes here.
};
```
前者调用后者。当然这个是简单的情况，我们可以很轻松的调换二者的顺序来解决

#### 发展
可是，往往事情没有这么简单
```cpp
class Two;

class One{
    int val;
    void fun1(Two two);
};

class Two{
    void fun2(One one);
};
```
嗯哼，这就形成了循环调用了。也好办，虽然不能调换顺序了，但是我们可以通过在```class One```定义之前先声明一下```class Two```来解决这个问题。
#### 升华
你可能发现上述代码仍然不能编译，为什么呢。
虽然在```One```之前声明了```Two```，但是编译器并不知道完整的```Two```的结构，所以无法给其分配内存。
完美的解决办法就是将参数的类型改为指针，这样编译器就不需要知道类的结构，而只需要知道一个等于当前操作系统字长的指针大小即可。
```cpp
class One{
    int val;
    void fun1(Two* two);
};

class Two{
    void fun2(One* one);
};
```
#### 结尾
没了。
